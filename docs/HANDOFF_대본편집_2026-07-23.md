# HANDOFF — 대본 씬 편집 + 파이프라인 정합 (2026-07-23)

화별 상세 대본 UX 개편 + 그에 맞춘 백엔드/프롬프트 수정. 대본 **포맷은 그대로**(LLM 입력 전용,
결정적 파서는 LLM 출력에만 붙음) — UI는 화면에서만 씬 단위로 파싱, 저장 시 원문 무손실 라운드트립.

## 프론트 (static/, 리로드 무관)
- **씬 카드 렌더**: `parseScriptSegments`/`renderScriptScenes` — `episode.script` 텍스트를 씬 세그먼트로
  분할해 카드로. 저장은 세그먼트 재조립(무손실). 도입/타이틀(preamble) 카드는 숨김.
- **대사/지문 구분**(`renderScriptMarkdown`): 콜론형 `이름: 대사`, 이름-헤더형(포맷 A), `(지문)` muted,
  `(Na)` 등 접미는 이름과 함께 강조, `*...*`로 감싼 인물+대사도 인식. 인물 이름은 성 뗀 변형까지 매칭.
- **씬 헤더 편집**: 카드 헤더의 로케이션/시간 인라인 input(`splitSceneHeader`로 `*N.…*` 포맷 보존), 볼드.
- **대본 편집 = 상시 편집**: 수정 버튼 제거, 씬 본문 textarea 상시 노출 + 자동저장(change, 재렌더 없이).
- **확정(잠금)**: "이 대본으로 확정" 버튼 → localStorage로 편집 잠금+읽기전용 스타일, "해제" 가능.
- **AI 생성 버튼 공통화**(`beginAiGen`): 생성 클릭 즉시 툴팁 닫힘 + "생성 중…" (4개 버튼 전부).

## 백엔드 (.py, 저장 시 uvicorn --reload)
- `openrouter_image.py` `_strip_voice_cue`: `resolve_element`에서 `(Na)/(V.O.)/(E)/(off)` 등 음성 큐
  접미 제거 후 이름 매칭 — 시간대 `(과거)` 구분은 유지. (내레이션 화자 참조/TTS 매칭 실패 버그 수정)
- `orchestrator.py` 캐릭터 초상: role을 정체성으로 반영(`_char_vibe`), 나이 정규화(`_age_en`: 50대→50s),
  `FACE_REF_APPEAL`(매력·조명 개선). 기능 제약(크롭·무의상·배경)·공유 `PORTRAIT_STYLE`은 유지.
- `studio.py` `_normalize_card_character`: 온보딩 피치카드 `role`("남 · 50대 회장")에서 성별·나이를
  gender/age 필드로 분리(create/update_project). "재벌 2세" 세대표기는 나이 오탐 안 함(N≥10만 나이).
- `orchestrator.py` `_ensure_v3_skeleton`: 대본 해시(`v3_skeleton_src`) 비교 → 대본이 바뀌면 스켈레톤 +
  `v3_scenes`(상세콘티) 자동 재생성. 같으면 캐시 재사용(이어서 만들기 정상).
- **프롬프트 포맷 인식**(스켈레톤 role + 상세콘티 `SCENE_BLOCKS_ROLE`): 씬헤더/대사/음성큐/지문 표기를
  명시적으로 읽게 하는 블록 추가.

## 분량
- 게이트 **90~110 → 90~120초** (`v3_schema.EPISODE_SECONDS_MAX = 120`).
- 목표치 **90 → ~100초** (대본 스펙/씬설계/스켈레톤 프롬프트, cw + storyboard 양쪽).
- ※ 대본 텍스트엔 초 없음 — 스켈레톤 단계가 초 배분·게이트 강제. 실측: 86289 ep1 = 100초(완주).

## 남은 것(미구현/주의)
- 확정 버튼은 UI 잠금까지만 — [확정]→분량 자동맞춤/게이트는 미구현.
- 스켈레톤 하드게이트(형식 `■ 씬N`·분량)는 여전히 LLM 출력 품질 의존.
